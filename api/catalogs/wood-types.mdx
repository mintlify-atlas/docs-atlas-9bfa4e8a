## Overview

The Wood Types API provides endpoints to manage wood type catalog entries in the furniture store system. Each wood type can have a name and optional description.

**Base URL:** `/wood-types`

## Endpoints

### List Wood Types

Retrieve all active wood types from the catalog.

```http
GET /wood-types/
```

#### Response

Returns an HTML page with the list of wood types. For API consumption, wood types are represented as:

<ResponseField name="id_wood_type" type="integer">
  Unique identifier for the wood type
</ResponseField>

<ResponseField name="name" type="string">
  Name of the wood type (max 100 characters)
</ResponseField>

<ResponseField name="description" type="string">
  Optional description of the wood type (max 255 characters)
</ResponseField>

<ResponseField name="active" type="boolean">
  Whether the wood type is active
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of creation
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

#### Example Request

```bash
curl -X GET http://localhost:5000/wood-types/ \
  -H "Accept: text/html"
```

#### Example Response

```json
[
  {
    "id_wood_type": 1,
    "name": "Oak",
    "description": "Strong hardwood with prominent grain",
    "active": true,
    "created_at": "2024-01-15T10:30:00Z",
    "updated_at": "2024-01-15T10:30:00Z"
  },
  {
    "id_wood_type": 2,
    "name": "Pine",
    "description": "Softwood with light color",
    "active": true,
    "created_at": "2024-01-16T14:20:00Z",
    "updated_at": "2024-01-16T14:20:00Z"
  }
]
```

### Create Wood Type

Create a new wood type entry in the catalog.

```http
GET  /wood-types/create
POST /wood-types/create
```

**GET** - Displays the creation form
**POST** - Processes form submission and creates the wood type

#### Request Parameters

<ParamField body="name" type="string" required>
  Name of the wood type
  
  **Validation:**
  - Required field
  - Minimum 3 characters
  - Maximum 50 characters
  - Must be unique
</ParamField>

<ParamField body="description" type="string">
  Optional description of the wood type
  
  **Validation:**
  - Maximum 200 characters
</ParamField>

#### Response

**Success:**
- Flash message: "Tipo de madera creado exitosamente"
- Redirects to `/wood-types/create`

**Error Cases:**

<ResponseField name="ConflictError" type="error">
  Returned when a wood type with the same name already exists
</ResponseField>

#### Example Request

```bash
curl -X POST http://localhost:5000/wood-types/create \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=Maple" \
  -d "description=Hard maple with fine grain pattern" \
  -d "csrf_token=your_csrf_token"
```

#### Error Response

```html
<!-- Flash message displayed -->
<div class="alert alert-error">
  Ya existe un tipo de madera con ese nombre
</div>
```

### Edit Wood Type

Update an existing wood type in the catalog.

```http
GET  /wood-types/<id_wood_type>/edit
POST /wood-types/<id_wood_type>/edit
```

**GET** - Displays the edit form pre-populated with current data
**POST** - Processes form submission and updates the wood type

#### Path Parameters

<ParamField path="id_wood_type" type="integer" required>
  Unique identifier of the wood type to edit
</ParamField>

#### Request Parameters

<ParamField body="name" type="string" required>
  Updated name for the wood type
  
  **Validation:**
  - Required field
  - Minimum 3 characters
  - Maximum 50 characters
  - Must be unique
</ParamField>

<ParamField body="description" type="string">
  Updated description for the wood type
  
  **Validation:**
  - Maximum 200 characters
</ParamField>

#### Response

**Success:**
- Flash message: "Tipo de madera actualizado exitosamente"
- Redirects to `/wood-types/<id_wood_type>/edit`

**Error Cases:**

<ResponseField name="NotFoundError" type="error">
  Returned when the wood type ID does not exist
  
  Flash message: "Tipo de madera no encontrado"
  
  Redirects to `/wood-types/`
</ResponseField>

<ResponseField name="ConflictError" type="error">
  Returned when the updated name conflicts with an existing wood type
</ResponseField>

#### Example Request

```bash
curl -X POST http://localhost:5000/wood-types/1/edit \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "name=Red Oak" \
  -d "description=North American hardwood with open grain" \
  -d "csrf_token=your_csrf_token"
```

---

### Delete Wood Type

Perform logical deletion of a wood type (marks as inactive).

```http
POST /wood-types/<id_wood_type>/delete
```

#### Path Parameters

<ParamField path="id_wood_type" type="integer" required>
  Unique identifier of the wood type to delete
</ParamField>

#### Response

**Success:**
- Flash message: "Tipo de madera eliminado exitosamente"
- Redirects to `/wood-types/`

**Error Cases:**

<ResponseField name="Exception" type="error">
  Generic exception handling for any deletion errors
  
  Displays exception message and redirects to `/wood-types/`
</ResponseField>

#### Example Request

```bash
curl -X POST http://localhost:5000/wood-types/1/delete \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "csrf_token=your_csrf_token"
```

## Data Model

### Wood Type Object

```python
{
  "id_wood_type": 1,
  "name": "Oak",
  "description": "Strong hardwood with prominent grain",
  "active": true,
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:30:00Z",
  "deleted_at": null
}
```

## Validation Rules

- **name**: Required, unique, min 3 characters, max 50 characters
- **description**: Optional, max 200 characters
- **active**: Boolean, defaults to `true`
- All endpoints require CSRF token for POST requests

## Error Messages

| Error | Message |
|-------|----------|
| Missing name | "El nombre del tipo de madera es requerido" |
| Name too short/long | "El nombre no puede exceder 50 caracteres" |
| Description too long | "La descripción no puede exceder 200 caracteres" |
| Duplicate name | "Ya existe un tipo de madera con ese nombre" |
| Not found | "Tipo de madera no encontrado" |
